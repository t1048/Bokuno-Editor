# 「僕の考えた最強のテキストエディタ」 基本設計書

## 1. システム概要

**コンセプト:** OSに深く統合され、極限まで「軽さ」と「速さ」を追求した次世代テキストエディタ＆高速検索ツール。

**対象OS:** Windows (※エクスプローラー拡張を優先するため。macOS/Linuxは段階的に対応可能)

## 2. 技術スタック

領域 | 採用技術 | 採用理由
**バックエンド (コア)** | Rust + Tauri | メモリセーフでネイティブに近い実行速度。Chromium非同梱による圧倒的な軽量化。
**フロントエンド (UI)** | React (TypeScript) | 宣言的なUI構築。仮想DOMによる効率的な描画と、豊富なエコシステムの活用。
**スタイリング** | Tailwind CSS | 軽量でユーティリティファーストなスタイリング。ビルド時のCSS最適化。
**エディタエンジン** | CodeMirror 6 | モジュール式で非常に軽量。Reactとの親和性も高く、拡張が容易。
**検索エンジン** | `ignore` (Rustクレート) | ripgrepのコアライブラリ。マルチスレッドでの超高速なファイル走査と正規表現検索。

## 3. アーキテクチャと責務分割

Tauriのベストプラクティスに従い、重い処理はすべてRust側に逃がし、Reactは「純粋な表示とユーザー操作の受け付け」に専念させます。

- **React (WebView側):**エディタ画面の描画（CodeMirrorのホスティング）検索UI（入力フォーム、結果一覧）の提供シンタックスハイライトの適用（CodeMirrorの言語パックを利用）
- **Rust (Core側):**OSのファイルシステムへのアクセス（読み込み・書き込み）指定ディレクトリに対するマルチスレッドGrep検索（`ignore`クレート）アプリケーションのウィンドウライフサイクル管理

## 4. 機能要件 (要件定義)

### 4.1. テキスト編集・表示機能

- **超軽量起動:** アプリケーション起動からエディタ画面表示までを瞬時に完了させる。
- **シンタックスハイライト:** ファイルの拡張子、またはマジックナンバーを解析し、適切な言語のハイライトを自動適用する。
- **巨大ファイル対応:** 数百MBのログファイル等を開いてもフリーズしないよう、Rust側でストリーム読み込み、あるいはフロントエンドで仮想スクロールを実装する。

### 4.2. OS統合機能 (エクスプローラー拡張)

- **「アプリ名で開く」:**対象: すべてのファイル (`*`)動作: エクスプローラーでファイルを右クリックした際、コンテキストメニューから直接本アプリで開く。
- **「Grep検索」:**対象: フォルダ・ディレクトリ動作: フォルダを右クリックし「Grep検索」を選択すると、本アプリの検索ウィンドウが立ち上がり、そのフォルダをルートとした全文検索モードに直接移行する。

## 5. 非機能要件

- **バイナリサイズ:** 10MB〜20MB以内を目指す（インストーラー含む）。
- **メモリフットプリント:** アイドル時 50MB 以下を目標とする。
- **インストール:** Windowsインストーラー（NSISまたはWiX）を作成し、インストール時に自動でレジストリ（右クリックメニュー）の登録を行うこと。

## 6. 開発マイルストーン

- **フェーズ 1: 基盤構築**Tauri + React + TypeScriptのプロジェクトセットアップ。RustとReact間のプロセス間通信（IPC）の疎通確認。
- **フェーズ 2: エディタ中核の実装**CodeMirrorの組み込み。Rust経由でのローカルファイルの読み書き機能の実装。
- **フェーズ 3: 高速検索の実装**Rust側でのGrep処理関数の実装。React側での検索UIと結果のハイライト表示機能の実装。
- **フェーズ 4: OS統合とパッケージング**起動引数（CLI引数）のパース処理（ファイルパスや検索対象ディレクトリの受け取り）。インストーラーの設定ファイル記述と、レジストリ登録スクリプトの作成
